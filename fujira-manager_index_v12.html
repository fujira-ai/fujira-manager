Problem

タスク管理が続かないのは、意思ではなく摩擦の問題です。

入力しづらい、見づらい、完了しづらい。その小さな面倒の積み重ねで、結局使わなくなります。フジラマネージャーはその摩擦を1つずつ削っています。

思いついた時に入れにくい

「明日17時に実家」「3月31日銀行処理」のような自然な言い方が通らないと、すぐ入力が止まります。

今日やることが埋もれる

未来タスクや期限なしタスクが混ざると、今やるべきものが見えなくなります。

通知が来ても行動に繋がらない

通知を見ても、そこから何を押せばよいか分からないと結局後回しになります。

How it works

登録 → 確認 → 完了まで、LINEの流れで自然につながります。

複雑な画面遷移はありません。自然文で送り、今日だけ見て、そのまま完了する。これを一番短く回せるようにしています。

1

自然文で登録

今日・明日・指定日・時刻・時間帯・食事語・タグまで、実際に打ちがちな言い方を広く吸収します。

2

見たい単位で確認

今日、一覧、期限なし、指定日、#タグ検索で、必要な範囲だけをさっと確認できます。

3

そのまま完了

今日画面や通知から完了クイックリプライで消せるので、入力して終わりになりません。

Features

ここまでに追加・改善された主な機能

正式版公開後も、実際の利用で見えた摩擦を基準に改修を重ねています。機能を増やすより、使いにくさを減らす方向を優先しています。

自然言語入力の強化

以下のような入力に順次対応しています。

  • 今日 / 明日 / 3月31日 / 3/31
  • 17時 / 19時半 / 20時10分 / 13:30
  • 明日17時に実家 / 今日20時10分ラインチェック
  • 3月31日銀行処理 / 3月31日は銀行処理 / 3月31日から作業
  • 今日夕方は打ち合わせ / 今日夕飯はしんぱちにする

表示と導線の改善

  • 一覧は1ページ5件、次/前、行動導線つき
  • 今日画面から完了1〜3をワンタップ
  • 期限なしタスクだけを見る専用コマンドを追加
  • 指定日検索（3月20日 / 3/20）に対応
  • タグ一覧（#仕事 など）もページング・完了対応

通知の改善

  • 朝通知・昼通知・前日通知
  • 予定時刻の10分前通知
  • 通知から完了導線とクイックリプライで行動へ接続
  • オンボーディングと1時間後フォローを最適化

バッティング確認

同日・30分以内に近い予定がある場合は、そのまま保存せず一度確認を出します。禁止ではなく、登録するかを自分で選べます。

#タグ機能

文末の #仕事、#私用 などを抽出して保存。#タグ名 を単独で送ると、そのタグだけの open タスクを確認できます。

使いやすさの細部改善

  • 時間帯で変わる「今日」コマンドの挨拶
  • 今日0件の時の行動導線
  • 完了後のリアクションと残件表示の自然化
  • 自然言語パースの失敗ログ可視化

フジラマネージャーが目指しているのは、タスク管理を頑張ることではありません。
思いついたら入る、今やる分だけ見える、終わったら自然に消える。
その繰り返しが続く状態を作ることです。

Update

ここまでの改修の方向性

機能を盛るのではなく、「詰まりやすい箇所」を順番に潰しています。

正式版までに整えた部分

  • オンボーディングと1時間後フォロー
  • 一覧のページングとクイックリプライ
  • リッチメニュー（今日 / 一覧 / ヘルプ）
  • 今日画面からの完了導線
  • ヘルプの行動ベース化

その後に追加した部分

  • 自然言語の精度強化（日時・時間帯・食事語・助詞・無セパレータ）
  • 指定日検索
  • 時間前アラート
  • 期限なし専用一覧
  • #タグ保存とタグ検索
  • 近接時間タスクの登録確認
